# Legacy app behavioral inventory

Audit scope: `app.html`, `public/app.js`, behavior-bearing selectors in `public/app.css`, and the active React/modular backend sources on branch `frontend/vite-react-rebuild`. This is an evidence inventory, not an implementation recommendation.

## Source-of-truth and method

Legacy behavior is concentrated in `public/app.js`, with initial DOM and a few inline handlers in `app.html`. The active frontend is `frontend/src/app/routes.tsx`, `App.tsx`, feature components, services, and state. The active backend runtime is `backend/app/main.py`; root `main.py` aliases that app for `uvicorn main:app`. `backend/app/api/v1/router.py` is an aggregation module but its own comment says it is not the runtime wiring target.

Search used literal endpoint/storage/event strings with `rg`, structural ownership by reading routes/components/services/endpoints, and the saved ast-grep configuration/rules where relevant. `frontend/dist`, datasets, logs, caches, `.env`, `main_ref.py`, and `arc_gateway` were excluded. The requested `$qma-codebase-navigation` skill was unavailable.

## Capability groups

### 1. Application bootstrap, live data, and view mode

- Trigger: DOM ready; page load; refresh control; basic/advanced toggle.
- Legacy evidence: `app.html` DOM-ready setup; `public/app.js` `loadLiveAnomalies`, `loadAgentRecommendations`, `loadProviders`, `loadMetrics`, `loadHealthInfo`, `setViewMode`; `/api/v1/live-anomalies`, `/api/v1/agent/recommendations`, `/api/v1/providers`, `/api/v1/platform/summary`, `/api/v1/config`.
- State/persistence: mutable globals for providers, anomalies, active query/provider, view mode; `localStorage.qma_view_mode`; live refresh uses `setInterval` (top-level timer) and request-specific timers.
- Visible outcomes: signal cards, provider selector/schema, metrics dropdown, health/config display, loading/empty/error states, basic versus advanced sections. Selector behavior includes `#stats-dropdown-btn/#stats-dropdown-panel`, `#advanced-details-toggle/#advanced-details-panel`, `.sidebar-panel-toggle`.
- Active trace: `AppPage.tsx` owns state/effects; `services/providers.ts`, `services/api.ts`; backend market/platform/health/provider route factories and their services/repositories.

### 2. Wallet connection, chain, balance, and wallet-bound session

- Trigger: connect/disconnect/menu/profile/funding actions; injected provider or mobile wallet modal.
- Legacy evidence: `connectWallet`, `disconnectWallet`, `restoreWalletSession`, `ensureArcTestnet`, `walletRequest`, `waitForReceipt`, mobile modal helpers; injected provider requests for accounts/chain/signing; gateway `/api/balance/:account` and `/api/wallet-status/:account`; `/api/v1/wallets/:account/session`.
- State/persistence: globals `connectedWallet`, wallet/provider status; `localStorage.qma_connected_wallet`; session token under `sessionStorage` key generated by `walletTokenCacheKey`; wallet events stored under generated local keys.
- Visible outcomes: connected address/menu role, chain warnings, balance/readiness, deposit/withdraw prompts, disabled actions, toast errors.
- Active trace: `WalletDropdown.tsx`, `FundArcWalletModal.tsx`, `walletStore.tsx`, `services/wallet.ts`, `walletProfileSession.ts`; backend wallets endpoints, gateway client/config, profile-token service. Risk: Critical/High.

## Timers and background work

Legacy has a top-level live refresh interval, quote-refresh timeout, invoice/gateway polling loops with 2-second sleeps, receipt polling, wallet request timeouts, toast removal timeouts, and focus/animation delays. Active React uses effects and local state; exact cleanup, cancellation, and interval cadence must be runtime-tested before cutover.

## Cross-cutting error/loading/empty behavior

Legacy normalizes API errors through `apiErrorMessage`, wallet errors through `normalizeWalletError`, and uses toasts plus inline payment/agent/profile states. Active components visibly model many loading/empty/error/disabled branches, but this audit did not execute browser flows; those branches remain verification items where exact copy, retry, cancellation, or cleanup matters.

## Known ambiguities

1. `render.yaml` starts `uvicorn main:app`; root `main.py` is a compatibility wrapper around `backend.app.main`, so deployment parity depends on wrapper hook behavior.
2. `backend/app/api/v1/router.py` says it is not wired, while `backend/app/main.py` registers router factories directly. Runtime route enumeration is required before cutover.
3. Persistence authority is environment-dependent; `SUPABASE_*` is configured in Render, but JSON fallback/state initialization must be verified from active config and repository selection. No data files were treated as evidence of authority.
4. The intended payment reference path is `docs/agent/PAYMENT_FLOW.md`, but only root `PAYMENT_FLOW.md` is present. Required follow-up: move or update the documentation path with explicit approval; this audit did not move files.

## Evidence policy update

The parity matrix deliberately separates implementation from verification. “Implemented” means an active source owner was located; it does not mean parity was executed. Any capability without a contract or browser run is `Static only` or `Needs verification`. Critical and High rows include exact legacy and active file/line anchors in `docs/agent/LEGACY_PARITY_MATRIX.md`.
